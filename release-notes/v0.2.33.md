# v0.2.33

## Workflow protocol-tool recovery

- Recover immediately when an active workflow CLI reports that `submit_phase_completion` or `request_workflow_route` is unavailable.
- Stop forwarding output from the stale CLI, rebuild it with the current workflow tool policy, and automatically replay the required protocol action.
- Prevent a missing workflow protocol tool from being converted into ordinary prose or an unrelated `AskUserQuestion`.
- Preserve the existing bounded retry behavior: a fresh CLI that still lacks the protocol tool produces a retryable visible error instead of looping indefinitely.

## Validation

- Targeted workflow runtime suites: 154 passed, 0 failed, 876 assertions.
- Coverage includes immediate submit recovery before stale prose/question output, route recovery, retry exhaustion, phase permissions, AskUserQuestion routing, and shipped workflow-pack behavior.
