- Keep missing workflow protocol tools and invalid protocol payloads on separate recovery paths.
- When `submit_phase_completion` or `request_workflow_route` fails input validation, return a structured correction instruction to the active model and require one retry of the same tool without restarting the CLI.
- Preserve bounded failure behavior: a second invalid protocol payload produces a retryable visible error and does not advance the workflow.
- Re-entered repair phases now clear stale completion, blocked, and artifact runtime fields so a Stage 5/6 → Stage 4 repair resumes with a fresh Stage 4 execution and subsequently follows the ordinary Stage 4 → Stage 5 transition.
- Normalize pending-confirmation summaries and recover desktop confirmation controls after a missing transition response.
