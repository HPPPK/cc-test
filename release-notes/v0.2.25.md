# cc-test v0.2.25

## Workflow protocol-tool binding reliability

This release fixes a race between desktop CLI prewarming and workflow activation.

### Highlights

- When a workflow begins while a prewarmed CLI is still starting, the runtime now waits for that startup to finish and immediately replaces it with a workflow-bound CLI.
- The replacement process receives the workflow session binding required to register `submit_phase_completion` and `request_workflow_route`.
- This prevents the first workflow phase from entering a CLI tool registry that lacks its required workflow protocol tools.

### Upgrade notes

- No workflow ZIP/template files changed.
- Existing in-progress CLI sessions should be restarted after upgrading so their tool registry is rebuilt with the active workflow binding.
- Windows installers and the GitHub Release are built remotely by GitHub Actions from tag `v0.2.25`.
