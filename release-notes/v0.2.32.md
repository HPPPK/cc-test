# v0.2.32

## Workflow blocked-phase recovery

- Allow a workflow phase that was explicitly submitted as `blocked` to request a confirmed `rework_current_phase` or `jump_to_phase` recovery route without bypassing `submit_phase_completion`.
- Preserve the controlled confirmation step for recovery routes and clear the blocked state only after the user confirms the approved route.
- Resume the selected implementation phase through the existing workflow runtime path, then return through normal linear phase progression after the repair completion is confirmed.
- Add regression coverage for Stage 5 and Stage 6 validation recovery to the implementation phase, structured AskUserQuestion route actions, and every shipped workflow pack's blocked recovery loop.
