# cc-test v0.2.22

## Workflow runtime routing and interaction reliability

This release makes workflow progression follow the server-approved route instead of assuming that every phase advances linearly.

### Highlights

- Structured workflow routing now supports confirmed non-linear repair loops, including returning from validation or preview phases to the correct implementation/fix phase when a real issue is found.
- `AskUserQuestion` choices with structured workflow actions can route to the selected phase and automatically resume execution. Users no longer need to type a separate “continue” message after a supported choice.
- Workflow confirmation cards use the server-provided pending target phase label and index. A request to return to Stage 4 is displayed as Stage 4, rather than being inferred as the next linear stage.
- Workflow turns that need a structured user interaction recover by requesting `AskUserQuestion` instead of silently ending as ordinary prose. Repeated protocol failures return a retryable workflow error rather than being presented as successful completion.
- Phase tool policy handling now preserves explicit tool names exactly and continues to enforce planning/debug-phase write restrictions at runtime.
- The shipped development, debug-repair, and feature-extension workflow packs have deterministic runtime regression coverage for route validation, confirmation, auto-resume, repair loops, permissions, pause/resume, invalid targets, and stale state versions. No workflow ZIP files were changed.

### Reliability and verification

- Stabilized Windows test execution for slow Git/worktree and verbose process-output scenarios without removing assertions.
- Fixed quality-gate output draining so verbose child commands cannot deadlock behind a full stdout/stderr pipe.
- Added filesystem fallback coverage for unavailable ripgrep/config startup timing.
- Full local quality gate passed: policy, workflow smoke, desktop lint/test/build, server tests, docs, coverage, native checks, persistence checks, and quarantine governance.

## Upgrade notes

- Existing workflow sessions keep their persisted state. Non-linear routes are applied through the runtime’s pending confirmation flow; a route is not silently committed before user confirmation when confirmation is required.
- This repository’s Windows package release assets are built by GitHub Actions from the `v0.2.22` tag. Local Windows packaging was attempted but this machine does not have the required Visual C++ Build Tools (`VC.Tools.x86.x64`).
