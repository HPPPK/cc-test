# v0.2.31

## Workflow protocol error detection

- Detect missing workflow protocol tools when the sidecar reports them through `tool_use_error` messages, not only through classic tool-result messages.
- Preserve the missing-tool signal until the terminal result so runtime rebind and automatic structured retry can run.
- Keep malformed inputs, stale state, permissions, and ordinary handler failures visible to the model as actionable tool errors instead of treating them as tool-registration failures.
