# v0.2.37

## Workflow confirmation safety
- 为每张阶段确认卡加入服务端确认单 ID，旧卡、延迟请求和重复点击不会推进到错误阶段。
- 服务端在确认单已被替换或状态版本过期时，先回传当前权威工作流状态，再返回明确错误；桌面端会据此移除旧卡。
- 正常线性推进继续使用 `submit_phase_completion`；返工、跳阶段、暂停和恢复仍走 `request_workflow_route`。

## Desktop and protocol alignment
- 桌面确认操作会携带确认单 ID 与权威 `stateVersion`，不再把内部 `revision` 错当作并发版本。
- 确认单变化时，本地等待锁和过期交互会立即失效，避免卡片停留或重复推进。

## Regression coverage
- 补充 REST、WebSocket、工作流包端到端和桌面状态回归测试。
- 服务端总测试与桌面全量覆盖率验证采用隔离执行，避免跨文件共享状态造成不稳定结果。

## Packaging
- Windows NSIS 安装包版本更新至 0.2.37。
