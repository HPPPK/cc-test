# v0.2.35

## Linear workflow continuation clarity

- Explicitly prohibit `request_workflow_route` from being used for ordinary linear progression after a normal phase completion.
- Add a runtime workflow-binding rule: after a Stage 4 repair, confirming the normal Stage 4 completion automatically enters Stage 5; do not create a redundant route request.
- When a same-target route is deduplicated against an existing pending completion, return a clear normal-completion message instead of implying that a second workflow route is waiting for confirmation.
